## 技術的制約

### ファイルシステム
1. trak-data/フォルダ構造
   - すべてのデータファイルを一元管理
   - フォルダ構成の厳格な維持

2. チケットファイル（trak-data/tickets/）
   - 形式: Markdown
   - エンコーディング: UTF-8
   - 命名規則の厳守

3. トラッキングファイル（trak-data/trackings/）
   - 形式: JSON
   - 厳格なスキーマ定義

4. 設定ファイル（trak-data/configs/）
   - 形式: JSON
   - スキーマによる型定義

5. テンプレートファイル（trak-data/templates/）
   - 形式: Markdown
   - 用途別のテンプレートファイル管理

### VS Code統合
- VS Code拡張機能として実装の可能性
- エディタとの円滑な連携
