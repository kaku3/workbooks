# 潜水艦ゲーム 仕様書 v4

## 1. ゲーム概要

| 項目 | 内容 |
|---|---|
| プレイ人数 | 3〜6人 |
| プレイ時間 | **10〜20分**（目標） |
| 通信方式 | PeerJS を使った P2P 通信（ホスト/クライアント構成） |
| 実装形態 | Static HTML + JavaScript（GitHub Pages 対応） |
| プレイ形式 | 同時プロット制（コマンドフェーズ→アクションフェーズ） |

### 世界観

深海を舞台にした情報戦。各艦長はソナーと機知で敵を探し、撃沈を目指す。
攻撃は位置を晒すリスク。補給ポイントを争いながら、収縮する戦場で最後の1艦になれ。
至近距離の接触はプライベートな死闘（ドッグファイト）に突入する。

### Apex的な駆け引きの再現

| Apex の要素 | このゲームの対応要素 |
|---|---|
| 足音・スキャンで敵を察知 | 攻撃・ソナーの副作用（音漏れ）で敵位置を推定 |
| 前方扇形センサー | **前方警戒システム**（進行方向3マスに反応があれば通知） |
| 至近距離のドッグファイト | **ドッグファイトモード**（接近した2者のみが互いの位置を知る） |
| 走り撃ち（ストレイフ） | 固定コマンド（←↑→）で自在に移動しながら攻撃カードを組み合わせ |
| 弾を全弾吐いてから逃げる | ターン燃料10を魚雷に全振り → 次ターン逃走全振り |
| 漁夫の利 | 2艦の戦闘の副作用が他者に漏れ、第三者に位置が割れる |
| リングによる強制移動 | 収縮ゾーンで外縁エリアがだんだん危険になる |

## 2. 燃料システム

### 2-1. 二層燃料ルール

```
■ 総燃料（タンク）: 100  ← ゲーム開始時の持ち燃料
■ ターン燃料上限   : 10  ← 1ターンに使える最大燃料
■ ターンで消費した分だけ総燃料が減少（残量は引き継ぐ）
■ 総燃料が 0 になると行動不能（固定コマンド含め一切使えない）
■ 燃料補給所で総燃料を +30 回復（上限100）
■ ドッグファイト中は移動コスト +1（機動が制限される）

手札＋固定コマンドから好きなだけ出せるが、
そのターンの合計燃料がターン上限を超えてはいけない。
```

> **設計意図**: 10ターン全力（10×10=100）で燃料が尽きる。
> 補給所へ行く判断が「攻めるか守るか」の戦略分岐になる。

### 2-2. ターンの行動例（ターン燃料10の場合）

| コンセプト | カードの組み合わせ | 合計燃料 |
|---|---|---|
| **全弾砲撃** | 魚雷(3) × 3本 + 固定前進(1) | 10 |
| **高速追撃** | ターボ移動(2) + 右横移動(2) + 魚雷(3) + 固定旋回(1)×2 | 10 |
| **見て撃って逃げる** | アクティブソナー(3) + 魚雷(3) + 固定旋回(1) + 固定前進(1)×2 | 10 |
| **最速逃走** | 緊急離脱(4) + ターボ移動(2) + 固定旋回(1) + 固定前進(1)×2 | 10 |
| **バフ→攻撃セット** | 強化ソナー(1) + アクティブソナー(3) + 連装砲準備(1) + 魚雷(割引=1) + 固定前進(1)×3 | 10 |
| **高出力ダッシュ** | 高出力推進(2) + ターボ移動(2=6マス!) + 左横移動(2=2マス!) + 固定前進(1)×3(各2マス!) | 10 |
| **省エネ偵察** | パッシブソナー(1) + 固定前進(1)×2 | 4（総燃料を温存） |

## 3. ボード構成と補給ポイント

### 3-1. マップ基本構成

- **サイズ**: 10×10 グリッド（座標 A1〜J10）
- **スタート配置**: ゲーム開始時にランダムで各艦を外縁付近に配置
- **情報の非対称性**: 自分の位置・向きは常に把握。他艦は副作用・索敵・警戒で推定

### 3-2. 補給ポイント（固定3箇所 + ランダム1箇所）

```
     A   B   C   D   E   F   G   H   I   J
  4  .   .   .  [燃] .   .  [修] .   .   .
  7  .   .  [弾] .   .   .   .  [?] .   .

[燃]=燃料補給所  [修]=修理港  [弾]=武器庫  [?]=ランダム補給（毎ゲーム変化）
```

| 補給ポイント | 効果 | 音漏れ |
|---|---|---|
| 燃料補給所（黄） | 行動フェーズ終了時にそこにいると総燃料 +30（上限100） | なし |
| 修理港（緑） | 行動フェーズ終了時にそこにいると HP +2（上限4） | なし |
| 武器庫（赤） | 行動フェーズ終了時にそこにいると任意山札から3枚追加ドロー | なし |
| ランダム（?） | ゲーム開始時に上記のいずれかに確定。内容は全員公開 | なし |

### 4-1. バトルロワイヤル式収縮ルール

```
ターン 1〜3:  10×10 全域が安全
ターン 4〜6:  8×8 が安全（外周1マスが危険海域）
ターン 7〜9:  6×6 が安全（外周2マスが危険）
```

- **危険海域**: アクションフェーズ開始時に HP -1
- 危険エリアのビジュアルは常に全員公開（赤オーバーレイ）
- 「次ターンから収縮」の予告は1ターン前から表示

## 5. 前方警戒システムとドッグファイトモード

### 5-1. 同マス衝突ルールの廃止

**旧ルール**: 他艦がいるマスには入れない  
**新ルール**: 同マスへの移動は可能。ただし以下の効果が発生する。

```
同マスに入ると → お互いの「正確な座標と向き」が2者間でのみ共有される
                  ドッグファイトモード に突入
                  他プレイヤーには この情報は届かない
```

### 5-2. 前方警戒システム

移動の意思決定を助けるため、**自艦の進行方向3マス以内に他艦がいる場合、自艦のみに警戒通知が届く**。

```
例: 艦が北向きで F5にいる場合
  警戒範囲: F4 / F3 / F2

  → F3 に別の艦がいれば「前方に反応あり（1〜3マス以内）」と表示
  → どのマスかは分からない（大まかな警戒のみ）
  → この通知は移動完了後・アクション解決時に発生
```

| 状況 | 表示 |
|---|---|
| 前方3マス以内に敵 | 「前方に反応あり」（緑ドット点滅） |
| 前方1マス（1歩先）に敵 | 「至近距離警報！」（赤ドット点滅） → ドッグファイト寸前 |
| 同マスに突入 | 自動でドッグファイトモード移行 |

> **ポイント**: 位置はバレないが「何かいる」ことはわかる。  
> 突っ込むか引くかの判断カードを使って読み合いが生まれる。

---

### 5-3. ドッグファイトモード

#### 発動条件

- 2艦が **同マスに入った**、または **互いに隣接（チェビシェフ距離 1）** した状態でターンが終わった場合

#### ドッグファイト中のルール

```
■ 2者間でのみ互いの座標・向きがリアルタイム公開（他プレイヤーは見えない）
■ 移動カードのコスト +1（追加燃料消費、機動の重さを表現）
■ 「緊急離脱」カードは通常コストで使用可能（脱出専用）
```

#### 魚雷の射線ルール（ドッグファイト中・同マス）

魚雷は「射線に捉える」ゲームデザイン。至近距離では射線が成立しない。

| 発射艦と対象の距離 | 命中判定 |
|---|---|
| 距離0（同マス） | **当たらない**（近すぎて射線が確保できない） |
| 距離1（ドッグファイト隣接） | **当たらない**（射線が成立しない） |
| 距離2以上 | 通常通り命中判定 |

> **設計意図**: 詰め寄れば安全だが、相手の前方3マス警戒に引っかかる。
> 「射線を作るために一度離れるか、真横に回り込むか」の判断が生まれる。
> ドッグファイトを抜ける唯一の武器は**緊急離脱**と**体当たりリスクの読み合い**。

#### ドッグファイト終了条件

- 2艦のチェビシェフ距離が **5以上** になった時点で解除
- 解除後は再び互いの位置が非公開になる

#### 第三者への影響（漁夫の利の設計）

```
ドッグファイト中の2艦:
  → 攻撃・移動の副作用（音漏れ）は通常通り全員に届く
  → しかし「2者が戦っている詳細」（座標）は届かない

第三者プレイヤー:
  → 副作用から「あのあたりで戦闘が起きている」ことは分かる
  → ピンポイントの座標は不明
  → タイミングを見て広域爆雷などで参入できる（漁夫の利）
```

#### ドッグファイトのビジュアル

- ドッグファイト中の2艦は、**自分の画面では位置が明示される**
- 他プレイヤーの画面では「その付近でドッグファイト中」のアイコンが1つ表示（座標は非公開）

## 6. カテゴリ別山札ドローシステム

### 6-1. 基本設計

手札のダブつき（移動ばかり、攻撃ばかり）を防ぐため、山札をカテゴリ別に4つに分割する。

```
山札A: 移動山札  （移動系カードのみ）
山札B: 索敵山札  （索敵系カードのみ）
山札C: 攻撃山札  （攻撃系カードのみ）
山札D: 特殊/バフ山札 （特殊系・バフ系カードのみ）
```

### 6-2. ドローフェーズのルール

ドローフェーズに毎ターン **2枚** ドローするが、どの山札から引くかを自由に選べる。

| 選択肢 | 内容 |
|---|---|
| A+A | 移動カードを2枚引く（逃走・補給ダッシュ重視） |
| B+C | 索敵1枚＋攻撃1枚（偵察攻撃セット） |
| A+D | 移動1枚＋特殊1枚（状況対応力重視） |
| C+C | 攻撃カードを2枚引く（全弾砲撃準備） |
| （任意） | 2枚をどの組み合わせでも自由 |

### 6-3. UIの操作感

```
ドローフェーズ開始
  │
  ├─ 画面に4つの山札アイコンが表示
  │    [移動] [索敵] [攻撃] [特殊/バフ]
  │
  ├─ プレイヤーが2回クリック（1枚目の山札 → 2枚目の山札）
  │    → 選んだ山札からカードが1枚ずつ引かれ手元にスライドイン
  │
  └─ ドロー完了 → コマンドフェーズへ
```

- 手札上限: 8枚。超える場合はドロー前に不要カードを捨てる
- 山札が切れた場合: 捨て札をシャッフルして補充（同カテゴリのみ）

## 7. ゲームの進行フロー

```
ゲーム開始
  │
  ├─ 初期配置フェーズ
  │    ランダム配置 + 補給ポイント位置を全員に公開
  │    各プレイヤーは各山札から1〜2枚ずつ選んで初期手札5枚をセット
  │
  ├─ [ターン繰り返し]
  │    │
  │    ├─ 1: 収縮チェック
  │    │    危険海域の艦は HP-1（アニメーションで演出）
  │    │
  │    ├─ 2: ドローフェーズ
  │    │    4種の山札アイコンから2枚を選んでドロー
  │    │
  │    ├─ 3: コマンドフェーズ（制限時間: 25秒）
  │    │    固定コマンド（←↑→）はいつでも使用可能（燃料のみ消費）
  │    │    手札＋固定コマンドをターン燃料上限内で選択・確定
  │    │    時間切れ → 確定済みのカードのみ実行
  │    │
  │    └─ 4: アクションフェーズ
  │         収縮ダメージ → 移動 → 補給取得判定 → 索敵 → 攻撃 → 特殊
  │         ドッグファイト判定 → 前方警戒通知 → ターンサマリ
  │
  └─ 勝利条件達成 → ゲーム終了
```

## 9. カード山札バリエーション

### 9-0. 固定コマンド（仮想カード）— 常時使用可能

手札を消費せず、**燃料のみ**で毎ターン使えるベースアクション。
デッキには含まれず、コマンドフェーズで画面上部に常時表示される。

| コマンド名 | 燃料 | 効果 |
|---|---|---|
| ← 左旋回 | 1 | 左に90°旋回（移動なし） |
| ↑ 前進   | 1 | 向いている方向に1マス移動 |
| → 右旋回 | 1 | 右に90°旋回（移動なし） |

> **設計意図**: 旧「前進」「方向転換」カードを固定化し、基本移動に手札を消費しない設計。
> 手札は**戦術カード**（高速機動・索敵・攻撃・特殊）に専念できる。
> 1ターンに複数回使用可能（燃料が許す限り）。高出力推進バフの影響も受ける。

### 9-6. バフ・デバフ系カード（7枚）

#### バフカード（5枚）

バフカードは**次のカード効果を強化する**「準備系カード」。このターン中に先に使って次の行動を強化する。

| カード名 | ID | 燃料 | 枚数 | バフ効果 | 有効範囲 |
|---|---|---|---|---|---|
| 高出力炉起動 | overcharge | 0 | 1 | **次のターン** のターン燃料上限 +4（最大14） | 次ターン1回 |
| 魚雷追尾装置 | torpedo_lock | 2 | 1 | **このターン次に使う魚雷** の射程 +3（9マスに） | 1発のみ |
| 強化ソナー | enhanced_sonar | 1 | 1 | **このターン次に使うソナー** の探知範囲 +2（パッシブ→5マス等） | 1回のみ |
| 装甲板展開 | armor | 2 | 1 | **次に受けるダメージ** を1軽減（0以下にはならない） | 次のダメージ1回 |
| 連装砲準備 | barrage_prep | 1 | 1 | **このターン次に使う攻撃カード** の燃料コストを -2（最低0） | 1枚のみ |

#### デバフカード（2枚）

デバフカードは**相手の特定行動を無効化または弱体化する**「妨害系カード」。
持続ターンを短く設定し、「長引くだけ」にならない設計とする。

| カード名 | ID | 燃料 | 枚数 | デバフ効果 | 持続 | 対策する脅威 |
|---|---|---|---|---|---|---|
| チャフ展開 | chaff | 1 | 1 | 自艦に使用。次の **2ターン**、誘導魚雷の追尾無効化（直線のみに） | 2ターン | 誘導魚雷への対抗手段 |
| 煙幕散布 | smokescreen | 2 | 1 | 自艦に使用。**このターンのみ**、アクティブソナーによる自艦検知を無効化 | 1ターン | 情報収集の遮断 |

> **デバフの設計思想**: ゲームを長引かせず、強力カードへの単純なカウンターとして機能させる。

#### バフ・デバフの使い方例

```
【バフ→攻撃コンボ】
  魚雷追尾装置(2) → 魚雷(3) → 射程9マスの超長射程魚雷！
  合計: 5燃料。残り5でさらに動ける。

【サポートターン設計】
  高出力炉起動(0) → 固定コマンドで偵察移動のみ → 次ターン燃料14で全力砲撃

【高出力ダッシュ】
  高出力推進(2) → ターボ移動(2=6マス!) → 固定前進(1=2マス!) × 2 → 合計10マス移動
  合計: 6燃料。爆速で補給ポイントへ。

【デバフ→逃走コンボ】
  煙幕散布(2) → 固定前進(1)×4 + 固定旋回(1) → ソナー無効で5マス移動
  合計: 7燃料。索敵から隠れながら移動。
```

## 10. コマンドフェーズのUI設計方針

### 10-1. カード選択の考え方

- **固定コマンド（←↑→）** が画面上部に常時表示。タップで即追加
- 手札は最大8枚が画面下部に扇状に並ぶ
- 各カードに **燃料コストをバッジ表示**
- バフカードは **金色のアウトライン** で区別（「このターン使うと強化できる」を視認）
- **燃料バー**: ターン燃料上限（10）と総燃料残量を並列表示
- 使用中カードを「実行列」に並べると燃料バーがリアルタイムで減る
- **25秒タイマー** を燃料バー上部に表示
- 機雷・追跡ブイ選択時はボード上に半透明パルスオーバーレイが表示され、直接クリックで座標指定

### 10-2. ドッグファイト中のUI変化

```
ドッグファイト中のレイアウト:
  ├─ 右上に「相手の座標と向き」をリアルタイム表示（自分だけ見える）
  ├─ 移動カードのコストが +1 でバッジ更新
  ├─ 「ドッグファイトモード」バナーが画面上部に表示（自分だけ見える）
  └─ 「緊急離脱」カードがハイライト表示（脱出の誘惑）
```

### 10-3. 前方警戒の表示

```
前方3マス以内に反応: ボード上の前方1〜3セルがうっすら橙色に染まる
前方1マスに反応: セルが赤くフラッシュ
同マスに突入する移動を選択中: 「ドッグファイト突入」警告ダイアログ（0.5秒）
```

## 11. 通信・ホスト制御仕様

- **ホスト**: ゲーム状態を一元管理。ドッグファイト判定も含めてアクション解決
- **クライアント**: 手札と行動選択のみローカル処理。送信は「コマンド確定」のみ
- **情報分離の原則**:
  - 索敵結果 → 発見したプレイヤーのみ受信
  - 副作用情報 → 該当する全員に送信
  - **ドッグファイト中の座標共有 → 当事者2名のみに送信**
- **タイムアウト処理**: 25秒経過で確定済みカードのみ実行（0枚なら何もしない）
- **ドローフェーズの同期**: 各プレイヤーがどの山札を選んだかはホストへ通知し、カード内容はローカルで処理
