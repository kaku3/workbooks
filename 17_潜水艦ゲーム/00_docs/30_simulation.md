# ゲームプレイ・シミュレーション v4.1

## セッション概要

| 項目 | 内容 |
|---|---|
| プレイヤー | 4人（アリス、ボブ、チカ、ダン） |
| モード | モード A：スタンダード |
| マップ | 10×10 グリッド |
| 初期HP | 4 |
| 総燃料 | 100（全員共通） |
| ターン燃料上限 | 10 |
| 手札上限 | 8枚（初期5枚） |
| デッキ | 42枚（移動12 / 索敵9 / 攻撃11 / 特殊+バフ+デバフ10） |
| 固定コマンド | ↑前進(2) / ←→旋回(3) 常時使用可（燃料のみ消費） |
| 重要ルール | 同マス・距離1では魚雷当たらない／カテゴリ別山札ドロー |

### v4 → v4.1 の主な変更点

| 変更 | 内容 |
|---|---|
| 固定コマンドコスト改訂 | 前進: 1→**2**、旋回: 1→**3** |
| 設計意図 | 旋回コストが重くなり「向き管理」の重要度UP。前進方向への直進がより効率的に |
| テンポへの影響 | 1ターンで旋回+前進3回=3+6=9燃料。直進5マス=10燃料。移動量は前進中心に絞られる |

### v3 → v4 の主な変更点（参考）

| 変更 | 内容 |
|---|---|
| 燃料システム刷新 | 総燃料100 + ターン上限10（毎ターンリセットではなくタンク消費制） |
| 固定コマンド追加 | ←↑→ が常時使用可能（手札不要・燃料のみ） |
| カード整理 | 前進・方向転換→固定化、巡航・急速推進→廃止、偵察魚雷・欺瞞信号・艦底放流・乗っ取り信号→廃止 |
| 新カード | ターボ移動(2)、高出力推進(2)、左横移動(2)、右横移動(2) |
| テンポ向上 | 「考える時間が長くなるカード」を排除し、アクション性を重視 |

## v4.1 ルール検証・ゲームデザイン評価

### 機能したポイント

| 要素 | 評価 | 詳細 |
|---|---|---|
| 固定コマンド（←↑→） | ◎ | 手札0枚でも移動可能。手札を戦術カードに専念できる基本設計は良好 |
| 旋回コスト3 | ◎ | 「一度向きが決まると変えにくい」という戦略性が生まれた。序盤の初期向き選択が重要に |
| 前進コスト2 | ○ | 直進効率の高さが際立つ。旋回より前進優先の戦術が自然に生まれる |
| 総燃料100制 | ◎ | ターンごとのリセットより戦略性UP。旋回コスト3との組み合わせで「節約vs行動」の判断が深い |
| ドッグファイト離脱 | ◎ | ターボ(2)+旋回(3)=5で離脱。燃料消費が明確になり「いつ逃げるか」の判断が明快 |
| ターボ移動 | ◎ | 燃料2で3マス移動は費用対効果抜群。旋回なしでそのまま飛ぶのが最も効率的 |
| 高出力推進 | ○ | 全移動2倍はロマン砲。14燃料ターンなら旋回(3)+前進(2)×5.5回に相当 |

### 課題・調整検討事項

| 課題 | 提案 |
|---|---|
| 旋回コスト3が重すぎる？ | アリスが5ターン連続で外縁に居続けた。**旋回コスト2**に戻して「旋回：前進＝1：1」の比率にするか、**旋回2 + 前進2**（同コスト）にする案 |
| 壁方向への魚雷不発が頻発 | **壁到達を事前警告**するUIを追加（コマンド選択時に「この向きでは魚雷は壁に当たります」表示） |
| 補給所+30の効果 | 総燃料100はまだ強力。**+20** or **+25**に下方修正を検討 |
| 前進コスト2で移動量減少 | 1ターン最大5マス（旋回なし）。v4の6〜8マスから減少したがテンポは問題なし |
| デッキ42枚でカード枯渇 | 4人×2枚/ターン=8枚/ターン。約5ターンで初回枯渇。**捨て札リシャッフルの頻度をモニタリング** |
| 総燃料表示のUI | 燃料バーに「ターン燃料/10」と「総燃料/100」を並列表示。**総燃料50以下で警告色**、**30以下で点滅** |

### v4 vs v4.1 テンポ比較

| 指標 | v4（旋回1/前進1） | v4.1（旋回3/前進2） |
|---|---|---|
| 1ターンの移動量 | 前進6〜8マス可能 | 前進4〜5マス（旋回なし）、旋回込み2〜3マス |
| 旋回の重み | 軽い（1コスト） | 重い（3コスト）→ 向き選択が戦略的に |
| 直進優先度 | 低い | **高い**（旋回3コスト節約のため） |
| 収縮ダメージとの相性 | 素早く脱出できる | 旋回コストで脱出が遅れる |
| コマンド思考時間 | 短い | やや長い（旋回を使うかの判断が増える） |
| 推奨調整 | — | **旋回コスト2への軽減** or **前進コスト1に戻す**を再検討 |

> **最終評価**: 旋回コスト3は「方向への縛り」が強烈すぎ、壁際プレイヤーが回復不能に陥りやすい。
> **旋回2/前進2** (同コスト) が最もバランス良い可能性。要追加テスト。
