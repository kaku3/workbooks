# パッケージ

## 世界観の確立

提案のアイデアを整理・統合する前に、「このゲームがどんな空気感のものか」を一行で定義しておく。  
バラバラに施策を打つと見た目がちぐはぐになるため、世界観が土台。

---

### 採用世界観：「深海の盤上、ソナーと魚雷の頭脳戦」

**コンセプト：タクティカル海戦ストラテジー**

> 10×10のグリッドに潜む潜水艦たち。誰がどこにいるかは分からない。  
> ソナーで探れば手がかりが得られる——だが音が漏れれば、次は自分が標的になる。  
> 燃料10で移動・索敵・攻撃を同時にプロットし、全員一斉にオープン。  
> 読み合いと決断力がぶつかる、3〜6人の海中タクティカルバトル。

**なぜこの世界観か：**

2025〜2026年のゲームトレンドを調査すると、このゲームに関連する以下の方向性が見える。

**■ 「不完全情報 × 戦略」ボードゲームの人気**
- *Star Wars: Rebellion*（BGG #10）——片方が秘密基地を隠し、もう片方が探す。「隠す側 vs 探す側」の非対称情報戦がこのゲームの構造と近い
- *Blood on the Clocktower*（BGG #69）——正体隠匿 × 推理。「誰が嘘をついているか」を読む構造は、「誰がどこにいるか」を読むこのゲームと通じる
- *The Crew: Mission Deep Sea*（BGG #43）——協力トリックテイキング・深海テーマ。「海底」テーマ × カードゲームの相性が実証された
- *Nemesis*（BGG #25）——SF × 疑心暗鬼 × サバイバル。裏切り要素なしでも「誰が何をしているかわからない緊張感」は人気

**■ 同時プロット制・リソース管理の定番人気**
- *Scythe*（BGG #26）——リソース管理 + エリアコントロール。限られた行動回数で何をするかの「同時選択のジレンマ」が高評価
- *Heat: Pedal to the Metal*（BGG #47）——手札管理 × 同時プロットのレースゲーム。「このターン何を出すか」の読み合いがパーティ層にも刺さっている
- *Root*（BGG #34）——非対称な目標・能力での対戦。多人数対戦で「誰が強いかをメタ的に読む」構造はこのゲームの漁夫の利設計と共通

**■ バトルロワイヤル的収縮メカニクス**
- Apex Legends の「足音・スキャンで察知 → 漁夫の利」構造がこのゲームの設計思想に直結
- ゲーム終盤に向けて密度を上げる「収縮ゾーン」はデジタル・アナログ問わずトレンド
- *Endeavor: Deep Sea*（BGG #85, 2024年）——海洋テーマ × 戦略ゲームの成功例。「海」はテーマとして健在

**■ ブラウザ対戦ゲームの空白地帯**
- 既存の潜水艦ゲームは **リアルシミュレーション路線（UBOAT、Sea Power）** か **アクション路線（Submarine.io）** に偏っている
- **「カード × 情報戦 × 同時プロット × ブラウザ即プレイ」という組み合わせは、明確な空白地帯**
- boardgamearena.com や screentop.gg のようなブラウザボドゲの需要増加もこの方向性を支持する

| 観点 | 採用理由 |
|------|----------|
| ターゲット感情 | 「見えない相手を推理する面白さ」と「攻撃すれば自分もバレるジレンマ」がゲームの芯と合致する |
| 差別化 | 潜水艦ゲーム = リアルシム or アクションの市場で、「カード × 情報戦 × ブラウザ」は唯一無二 |
| 実装コスト | 深海のダーク基調は色数が少なく描画コストが低い。濃紺 + 発光色で雰囲気が成立する |
| SNS映え | ダーク × ネオン系のUIはスクショ映えが良く、ソナー波紋のGIFやプレイ結果がシェアされやすい |

**ビジュアルの基準軸：**

- 背景：漆黒〜深海ネイビー（#050d1a〜#0a1628）
- アクセントカラー：**ソナー/索敵 = シアン（#00e5ff）**、**攻撃/爆発 = 琥珀〜赤（#ff6600〜#c0392b）**、**補給 = 黄緑（#aaffcc）**
- 艦体：明るいシアン（自艦）、発見時は各色（赤/黄/紫/オレンジ）
- グリッド線：半透明の青緑（仕様書の `rgba(0,200,200,0.15)` と整合）
- フォント：見出しにシャープなゴシック、ログに等幅フォントで「戦術モニタ感」を演出
- テイスト：「ソナー画面」「海戦ストラテジーゲームのHUD」「ボードゲームのクリーンなUI」を参考にする

## パッケージ提案（実装難易度: 低→高 順）

### Lv.1 ［テキスト変更のみ・0コスト］

#### 1-A. キャッチコピーの差し替え

「3〜6人で遊べる同時プロット制ゲーム」は説明であって、**「手に取る動機」ではない**。  
「見えない敵を探し、撃てば自分もバレる」というジレンマへの期待に言葉を変える。

| 案 | コピー | 狙い |
|----|--------|------|
| 短文 | 「撃てば、バレる。沈黙か、魚雷か。」 | 一行でジレンマを伝える |
| 感情訴求 | 「深海で響くのは、ソナーの反響と自分の鼓動だけ——3〜6人で沈め合う、情報戦すごろく。」 | 「疑心暗鬼」体験を想像させる |
| ゲーム性訴求 | 「見えない敵を探し、撃ち、逃げる。燃料10で全てを賭ける同時プロット潜水艦戦。」 | 仕組みを知りたい層に刺さる |

#### 1-B. SNSシェア定型文の設計

シェアボタンの文言を事実のログではなく、**感情のログ**にする。

```
勝利時（最後の1艦）：
「最後の1艦になった。ソナーで見つけて、魚雷で沈めて、補給で逃げ切った。 #潜水艦ゲーム」

ドッグファイト勝利時：
「至近距離で見つかった瞬間、冷や汗が出た。緊急離脱を切って逃げた先で魚雷を叩き込んだ。 #潜水艦ゲーム」

撃沈時：
「F列に魚雷を撃ったら、G列から報復が来た。誰だお前。 #潜水艦ゲーム」
```

#### 1-C. ゲームログの演出テキスト

「Aさんが魚雷を発射した」→「【発射警報】A艦長——F列方向に魚雷射出。反響音が全域に伝播」  
「チカがパッシブソナーを使用」→「【索敵】C艦——パッシブソナー展開。半径3マスを走査中...」  
ログが「CIC（戦闘情報センター）の通信」になるだけで、緊張感の質が変わる。

| 場面 | 現在のログ例 | 演出ログ |
|------|------------|---------|
| 魚雷発射 | 「Aが魚雷を発射」 | 「【射撃警報】A艦——F列射線に魚雷1本射出。反響、全域に拡散」 |
| 命中 | 「BにHit」 | 「【着弾確認】B艦に魚雷命中。船殻損傷——HP残 2」 |
| ソナー | 「Aがソナーを使用」 | 「【音響走査】A艦——アクティブソナー発振。位置が全域に漏洩」 |
| ドッグファイト突入 | 「AとBが接触」 | 「【至近接触】A艦-B艦——ドッグファイトモード突入。相互探知開始」 |
| 補給獲得 | 「Aが燃料補給所に到達」 | 「【補給完了】A艦——D4燃料補給所に接舷。次ターン燃料+5」 |
| 収縮警告 | 「次ターンから収縮」 | 「【海域警報】外周エリア——危険海域指定。次ターンよりHP-1」 |
| 撃沈 | 「Bが沈没」 | 「【轟沈】B艦 ████ 全信号ロスト。残存艦 3」 |

#### 2-B. フォントの差し替え

```css
  box-shadow: 0 0 20px rgba(0, 229, 255, 0.4);
  border-color: #00e5ff;

/* カテゴリ別 */
.deck-move   { border-top: 3px solid #00bcd4; }
.deck-recon  { border-top: 3px solid #00e5ff; }
.deck-attack { border-top: 3px solid #ff6600; }
.deck-special { border-top: 3px solid #ffe066; }
```

### Lv.4 ［体験設計 → UXの磨き込み・中〜高コスト］

#### 4-A. ゲーム開始演出の追加

ゲーム開始時に2〜3秒の軽い演出を入れ、対戦の緊張感を高める。

```
【演出シーケンス】
  0.0s: 画面が暗転
  0.5s: 画面中央にシアンのテキストがフェードイン
        「全艦配置完了——索敵開始」
  1.5s: ソナーのピン音（ピーン）が1回鳴る（任意）
  2.0s: テキストがフェードアウト
  2.5s: マップがフェードインし、自艦の初期位置と補給ポイントが表示される
  3.0s: UIが全て表示。ゲーム開始
```

コード的には `overlay.css` + `overlay.js` へのフェードインアニメーション追加のみ。

#### 4-B. ドッグファイト突入演出

至近遭遇の緊張を最大化する演出。仕様書の `dogfightEntry` アニメーションを拡張。

```
【ドッグファイト突入シーケンス】
  0.0s: 画面全体が一瞬赤くフラッシュ
  0.1s: ダメージシェイク（damageShake を0.3秒）
  0.3s: 「⚠ ドッグファイトモード」バナーが上部からスライドイン（赤背景 + 白文字）
  0.5s: 右上に敵艦情報パネルがフェードイン
  1.0s: バナーが落ち着いた色（#1a0808 × 70%透過）にフェード
```
