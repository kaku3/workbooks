## M-01: プレミアム割引、なぜか効かないユーザーがいる

**依頼内容**

「佐藤花子さんからクレームが来た。プレミアム会員なのにカートで割引が表示されない」

**確認してほしいファイル**

- `db/users.json`
- `public/cart.html`（`// プレミアム割引` のあたり）
- `app.js`（`// プレミアム割引` のあたり）
- `public/mypage.html`（`// プレミアム判定` のあたり）

**設問**

1. `db/users.json` の各ユーザーのデータ構造を比較してください。フィールド名に違いはありますか
2. `cart.html` と `app.js` と `mypage.html` では、それぞれどのフィールドでプレミアム判定していますか
3. 佐藤花子のカート割引が効かない理由を、コードを根拠にして説明してください
4. この問題を修正するとしたら、どこをどう直しますか。またどこを変えると他への影響が最小ですか

## M-02: トークンの実装が2種類ある

**依頼内容**

「ログインしても一部の API で401が返ってくることがある。トークン周りを確認してほしい」

**確認してほしいファイル**

- `app.js`（`// ログイン` のあたり）
- `utils.js`（`generateToken` / `getUserIdFromToken` のあたり）

**設問**

1. `app.js` のログイン処理でトークンはどのように生成されていますか
2. `utils.js` の `generateToken` はどのようにトークンを生成しますか
3. `utils.js` の `getUserIdFromToken` は、`app.js` が生成したトークンを正しく解析できますか。コードを根拠に説明してください
4. この問題をどう整理しますか（どちらを残すか、どのように統一するか）

## M-03: 在庫が減らない注文処理

**依頼内容**

「注文は通るのに在庫が減っていないと言われた。確認してほしい」

**確認してほしいファイル**

- `app.js`（`POST /api/orders` のあたり）

**設問**

1. `POST /api/orders` の処理を追ってください。注文確定時に在庫を減らす処理は呼ばれていますか
2. `POST /api/stock/:productId/decrement` というエンドポイントは存在しますか。それはどこから呼ばれますか
3. この状態のまま運用を続けた場合、どのような問題が起きますか
4. 修正するとしたらどのようなアプローチを取りますか。また修正前に確認・検討すべきことは何ですか
