# B-02: `utils.js` と `common.js`、どっちが正しい？ — 模範解答

## 設問 1：金額フォーマット関連の関数

2箇所に存在する。

| ファイル | 関数名 |
|---------|--------|
| `utils.js` | `formatPrice(amount)` |
| `public/common.js` | `formatPrice(amount)`（同名の関数が存在） |

同様に `formatDate` も両ファイルに重複している。

> **採点ポイント**：2箇所あることを明示できているか。「utils.js にある」だけでは △。

## 設問 2：どちらを修正するか

**両方修正する必要がある。**

- `utils.js` はサーバーサイドの Node.js コードで使われる（`require` で読み込む）
- `public/common.js` はブラウザ（HTML ファイルの `<script>` タグ）で読み込まれる
- この2つは独立しており、片方を直してももう片方には反映されない

変更内容は：
```js
// 両ファイルの formatPrice を修正
function formatPrice(amount) {
  return '¥ ' + Number(amount).toLocaleString() + ' '  // ← スペースを追加
}
```

ただし、**修正前に「どの画面が utils.js を使っていて、どの画面が common.js を使っているか」を確認する**必要がある。この確認を怠ると設問 3 の問題が起きる。

> **採点ポイント**：「両方直す」と「なぜ両方直す必要があるか」の説明が必要。「utils.js だけ直す」は △。

## 設問 3：「一部の画面だけ反映されない」原因

最も考えられる原因：**片方のファイルだけ修正して、もう片方を修正し忘れた。**

- `utils.js` のみ修正 → サーバーサイドレンダリングや、utils.js を使っている箇所は更新されるが、common.js を使っているブラウザ側の画面は変わらない
- `common.js` のみ修正 → ブラウザ画面は更新されるが、サーバー側の金額表示は変わらない

他の可能性：
- ブラウザキャッシュが残っている（hard reload で解決）
- 一部の画面だけ `common.js` を読み込んでいない

> **採点ポイント**：「重複しているから」という原因を正確に説明できているかを見る。

## 設問 4：同じ関数が複数ファイルに存在する原因

- フロントエンドとバックエンドで「共有できる JS ファイル」の仕組みがない（node_modules 経由のパッケージ化をしていない）
- 「サーバーで使う関数」と「ブラウザで使う関数」を分ける設計指針がなかった
- 最初に `utils.js`（サーバー用）を作った後、フロントでも同じ関数が必要になったとき「コピーした方が早い」と判断した
- コードレビューで「共通化すべき」と気づかなかった / 指摘されなかった
