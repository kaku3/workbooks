# `50_evaluation/10_samples/` — 使い方サンプル

このフォルダは **「採点フローの使い方をつかむための実例」** です。
B-01（初級）1問分の、薄い回答・良い回答それぞれの完結した採点セットが入っています。

## フォルダ構成

```
10_samples/
├── README.md           ← このファイル
├── 10_薄い/            ← 「それっぽいが浅い」回答のサンプル一式
│   ├── 30_answers/     ← 受験者の回答（テンプレをコピーして使う）
│   │   └── 10-B01_utils肥大化.md
│   ├── 31_reviews/     ← AI 採点結果（30_answers/ と同名ファイルが並ぶ）
│   │   └── 10-B01_utils肥大化.md
│   └── summary.md      ← 全設問完了後のサマリー
└── 11_良い/            ← 合格ラインの回答のサンプル一式
    ├── 30_answers/
    │   └── 10-B01_utils肥大化.md
    ├── 31_reviews/
    │   └── 10-B01_utils肥大化.md
    └── summary.md
```

## 実際の採点での使い方

### ① 受験者フォルダを準備する

```
{受験者名}/
├── 30_answers/   ← プロジェクトの 30_answers/ をまるごとコピーして受験者の回答を入れる
├── 31_reviews/   ← 最初は空（AI 採点結果をここに保存していく）
└── summary.md    ← 全設問完了後に生成
```

### ② ステップ 1：全設問を一括採点する

`50_evaluation/score_prompt.md` の **「ステップ 1：一括採点プロンプト」** を使います。

1. プロンプト本文を Copilot チャット（Agent モード推奨）に貼り付ける
2. 以下の 4 つを `#file:` で添付して送信する：
   - `#file:{受験者名}/30_answers/`
   - `#file:40_reference-answers/`
   - `#file:20_questions/`
   - `#file:10_sample-ec/`
3. AI が設問ごとに採点結果を出力するので、`{受験者名}/31_reviews/{同名ファイル}.md` に保存する

→ サンプルの出力イメージ：`10_薄い/31_reviews/10-B01_utils肥大化.md`

### ③ ステップ 2：全設問完了後にサマリーを生成する

`50_evaluation/score_prompt.md` の **「ステップ 2：サマリープロンプト」** を使います。

1. `31_reviews/` 内の全ファイルの内容をまとめて AI に渡す
2. 出力を `{受験者名}/summary.md` に保存する

→ サンプルの出力イメージ：`10_薄い/summary.md`

## 薄い回答と良い回答の違い

`10_薄い/30_answers/` と `11_良い/30_answers/` を読み比べてください。
同じ B-01 への回答でも、採点結果（`31_reviews/`）と `summary.md` がどう変わるかが確認できます。

## 採点者としての最終判断

AI の採点はドラフトです。`31_reviews/` 内の各ファイル末尾にある「採点者メモ欄」で人が上書きしてください。
