# A-02: クーポンはフロントだけで計算されている — 模範解答

## 設問 1：バグの原因

> **採点の観点**：「フロントで計算している」だけでなく、「サーバーに送られていない」→「サーバーが検証できない」という流れを追えているかを見る。

**現状のフロー**：

1. `checkout.html` が `GET /api/coupons/:code` でクーポン情報を取得
2. フロントエンドで割引額を計算
3. `POST /api/orders` に送るリクエストボディは `{ address: address }` のみ → **クーポン情報・割引情報はサーバーに送っていない**
4. サーバーは受け取った値をそのまま注文として保存

→ クーポンを適用したことがサーバーに伝わっておらず、注文金額が割引されないまま確定される。

**さらに深刻な問題**：

クーポン割引がフロントのみで計算・表示されており、サーバー側での検証がない。  
ブラウザの開発者ツールでリクエストを改ざんすれば、任意の金額で注文を確定できる。  
また、クーポンコードの有効性（期限切れ・使用済み・不正なコード）もサーバーで検証されていない。

**採点基準**
- ◎：「クーポン情報がリクエストに含まれていない」→「サーバーが検証できない」→「改ざんも可能」の流れを説明できている
- ○：「フロントだけで計算されていて、サーバーに伝わっていない」を説明できている
- △：「フロントで計算している」とは言っているが、なぜバグになるかの説明がない
- ×：バグの原因を特定できていない

## 設問 2：設計の問題点と修正方針

> **採点の観点**：2 つの方式のトレードオフを正確に理解し、どちらを選ぶかの根拠を説明できるかを見る。

**方式 A：フロントで計算してサーバーに金額を送る（現状）**

- メリット：フロント実装が単純。サーバーに複雑なクーポン検証ロジックが不要
- デメリット：リクエスト改ざんで任意金額で注文できる。セキュリティ上採用できない

**方式 B：クーポンコードをサーバーに送ってサーバー側で検証・計算する（推奨）**

- メリット：サーバーが金額の最終決定権を持つため改ざんできない。有効性チェックも確実に行える
- デメリット：サーバーにクーポン検証・計算ロジックが必要。フロントのプレビュー表示と金額が一致しない場合の考慮が必要

**推奨：方式 B**

決済金額はサーバーが決定するのが原則。フロントでのプレビュー計算は「表示のため」として残してよいが、実際の注文金額はサーバーで再計算する。

```js
// POST /api/orders のリクエストボディ（修正後）
{ address: "東京都", productId: 1, quantity: 1, couponCode: "SAVE10" }

// サーバー側（app.js）でクーポン検証・金額再計算
if (couponCode) {
  const coupon = COUPONS.find(c => c.code === couponCode)
  if (!coupon) return res.status(400).json({ error: '無効なクーポンコードです' })
  total = Math.floor(total * (1 - coupon.discountRate / 100))
}
```

**採点基準**
- ◎：2 方式のメリット・デメリットを正確に説明し、選択の根拠（サーバーが最終決定権を持つ原則）を述べられている
- ○：どちらを選ぶかの判断とその理由がある
- △：「サーバー側で検証すべき」とは言っているが、方式の比較や根拠がない
- ×：「フロントで計算して送ればいい」（方式 A を改善なしで採用）

## 設問 3：影響するファイル

> **採点の観点**：「checkout.html か app.js の片方だけ直せばいい」という思い込みがないかを見る。

**両方の変更が必要**。片方だけでは済まない。

| ファイル | 変更内容 |
|---------|---------|
| `app.js` | クーポンコードの受け取り・検証・金額再計算処理を追加 |
| `public/checkout.html` | リクエストボディに `couponCode` を含めるよう変更 |

さらに確認すべきファイル：
- `db/coupons.json`（またはコード内の `COUPONS` 定数）：使用済みクーポンのチェックを追加する場合、クーポンの状態管理が必要
- 注文確定後の領収書表示や確認画面がある場合は、割引後の金額表示も修正対象

**採点基準**
- ◎：両ファイルの変更内容を具体的に説明し、関連ファイルも挙げられている
- ○：`app.js` と `checkout.html` の両方が必要と説明できている
- △：「片方だけ直せばいい」という判断をしている
- ×：影響ファイルを挙げられていない
