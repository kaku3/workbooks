## S-02: 図を描いてみよう

**状況**

S-01 で実装した設計を、チームメンバーにレビューしてもらうことになった。  
口頭で説明しようとしたが、「どこが何を呼んでいるか」がうまく伝わらなかった。

**Step 1**

PlantUML を使って、S-01 で実装した注文キャンセル機能の処理フローを図にしてみよう。  
まず AI に頼んでみよう。

```
S-01 で実装した注文キャンセル機能の処理フローを PlantUML のシーケンス図で書いて
```

**Step 2**

生成された PlantUML を [PlantUML : VS Code Extension](https://marketplace.visualstudio.com/items?itemName=jebbs.plantuml) で確認してみよう。  
図を見ながら、コードと見比べてほしい。

**設問**

1. AI が生成した PlantUML の図と、実際のコードは一致しているか。  
   ズレている部分があれば、どこがどうズレているかを説明してほしい。
2. 図を見て初めて気づいたことはあるか。コードだけを読んでいたときと、何が違ったか。
3. この図を見た別の人が「在庫を戻す処理はどこで呼ばれているか」と聞いてきたとき、図を使って説明してみよう。

> **ヒント**：PlantUML は Markdown と同じようにテキストで書ける図ツールです。  
> Git で管理でき、AI に読ませることもできます。  
> 「図を描くツール」というより「設計の記録をテキストで残すツール」と考えてみてください。
