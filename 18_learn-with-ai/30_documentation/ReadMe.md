# 設計ドキュメント編（Design Documentation）

## 学べること

AI と一緒に機能を実装しながら、**設計を図で残す・更新する・引き継ぐ**一連の流れを体験します。

PlantUML を使ったシーケンス図の作成から、コードと図の乖離への対処、AI への引き継ぎまでを通じて  
「設計ドキュメントを残すことの意味」を自分の言葉で説明できる力を養います。

| テーマ | 内容 |
|--------|------|
| AI への設計説明 | 曖昧な依頼から実装し、設計を文章化する限界を体験する |
| 図の作成 | PlantUML でシーケンス図を描き、コードとの照合を行う |
| 設計変更への追従 | 要件追加に合わせて図を更新し、ドキュメント維持コストを実感する |
| AI への引き継ぎ | 図＋文章＋コードを渡して AI に続きを頼み、何が伝わるかを検証する |

---

## 使い方

1. `10_sample-ec` を clone してコードを把握する
2. `10_questions/` の問題ファイルを読む（S-01 から順番に取り組む）
3. `20_answers/` に自分の回答を書く（ファイルは設問ごとに分かれています）
4. `30_reference-answers/` で模範解答を確認する
5. `40_evaluation/score_prompt.md` のプロンプトを使って Copilot に採点させる

> AI ツールの利用は自由です。ただしすべての設問で「なぜそう判断したか」を自分の言葉で書いてください。
