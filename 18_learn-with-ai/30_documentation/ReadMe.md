# 設計ドキュメント編（Design Documentation）

## 学べること

「何も知らないプロジェクトにアサインされたとき、どうやって全体像を把握するか」を体験する**チュートリアル**と、  
AI と一緒に機能を実装しながら**設計を図で残す・更新する・引き継ぐ**一連の流れを体験する問題セットの、2 部構成です。

| テーマ | 内容 |
|--------|------|
| 全体像の把握 | サイトマップ・機能仕様書・ER 図・画面仕様書・シーケンス図をAIで作りながらプロジェクトを理解する |
| AI への設計説明 | 曖昧な依頼から実装し、設計を文章化する限界を体験する |
| 図の作成 | PlantUML でシーケンス図を描き、コードとの照合を行う |
| 設計変更への追従 | 要件追加に合わせて図を更新し、ドキュメント維持コストを実感する |
| AI への引き継ぎ | 図＋文章＋コードを渡して AI に続きを頼み、何が伝わるかを検証する |

---

## 使い方

1. `sample-ec` を clone してコードを把握する

   ```bash
   # SSH
   git clone git@github.com:kaku3/workbooks.git
   # または HTTPS
   git clone https://github.com/kaku3/workbooks.git

   cd workbooks/18_learn-with-ai
   ```

2. **チュートリアル（T-01〜T-05）** から始める場合：  
   `10_questions/` の `11-T01_サイトマップを書こう.md` から順番に取り組む

3. **設計問題（S-01〜S-04）** から始める場合：  
   `10_questions/` の `21-S01_AIに設計を伝えてみよう.md` から順番に取り組む

4. `20_answers/` に自分の回答を書く（ファイルは設問ごとに分かれています）
5. `30_reference-answers/` で模範解答を確認する
6. `40_evaluation/score_prompt.md` のプロンプトを使って Copilot に採点させる

> AI ツールの利用は自由です。ただしすべての設問で「なぜそう判断したか」を自分の言葉で書いてください。

## チュートリアル一覧（T系：全体像を把握する）

チュートリアルは「何も知らないプロジェクトへの入り方」を体験する内容です。  
はじめてこの教材に取り組む場合、または Essential の問いで何をすればいいか分からない場合は、こちらから始めてください。

| 問題 | タイトル | テーマ |
|------|----------|--------|
| T-01 | サイトマップを書こう | 実際に動かして全ページを把握・サイトマップ生成 |
| T-02 | 機能仕様書をAIに書かせよう | 情報量によるAI出力の変化・仕様書の信頼性 |
| T-03 | データ設計図をAIに書かせよう | ER 図の生成・データ構造の理解 |
| T-04 | 画面仕様書をAIに書かせよう | スクリーンショット活用・表示条件の整理 |
| T-05 | PlantUMLのシーケンス図をAIに書かせよう | 処理フローの可視化・全体振り返り |
